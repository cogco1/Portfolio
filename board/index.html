术语 Glossary

真源 = docs/理论(记法) + 术语对照 + 提案卷 · 判决时更新

理论框架 Theory — 论文的骨头(记法以理论卷为准)

MDP · 马尔可夫决策过程Markov Decision Process

全项目的理论底座:建筑构成 = 一串决策的轨迹 p(τ)=p(a₀…a_T|s₀),建筑只是轨迹终点 B=f(τ)。与其学"条件 C 下采样成品 B",不如构造能稳定推进、限制组合爆炸、记录承诺、允许局部修订的设计状态系统。"建筑不是被采样出来的,而是在连续承诺中成为现实。" 理论卷 §0

POMDP · 部分可观测 MDPPartially Observable MDP

真实设计天然有隐藏意图——"这里不能放柱子"的真正原因可能是没说出口的视线、仪式序列或未来立面构想,所以天然是部分可观测的。ArchFlow 的对策不是硬学,而是把意图显式化 → 见「构造马尔可夫性」。 理论卷 §3.3

构造马尔可夫性Constructed Markovianity

不是发现设计过程是马尔可夫的,而是把它构造成马尔可夫的:隐藏意图编译为显式状态变量(前提 premises:parti / function_intent / tradition,铁律11 强制每个决策必读),让"只看当前状态就能走下一步"成为工程事实。 premises.py · 理论卷 §3

状态编译 S_tState Compiler

把历史、提案、验证结果编译成最小充分状态 S_t=(G,C,L,O,U) + 义务账本 𝒪_t;完整思考历史反而引发漂移,编译态才可控。代码 = 黑板(槽位读写集+前提)。 blackboard/ · 谱系:Hayes-Roth 1985 · Hearsay-II

熵预算Action Budgeter

每步条件熵 H(A_t|S_t)≈log₂|A_valid| 压在预算带 [H_min,H_max] 内:掩码先把菜单缩到合法集 A_valid,策略只在合法项里选。代码 = MenuSpec(预掩码菜单,三真菜单已通)。 policy.py · 谱系:SPRING(AIJ 2024) · shielded RL

D⁺(X) · 依赖闭包Dependency Graph

决策 X 的全部下游 D⁺(X)={Y|X⤳Y};修订成本 C_revision ∝ |D⁺(X)|——改一个决策的代价由依赖闭包决定,不由结果规模决定(repair locality 的测量面)。代码 = 读写集 + downstream_of()。 谱系:TMS(Doyle 1979) · de Kleer 1986

Selective Uncommit选择性修订 · P3.1

invalidate(D⁺(X)) → 局部重规划,而非整图重生:只解除被波及的承诺,其余照旧。修改不可逆的三个原因(下游扩散/编译丢史/路径依赖)正是它要绕开的。施工序 P3.1 在建,全案最紧赌注(9 月检查点)。 理论卷 §5–6

RAD · 检索增强决策Retrieval-Augmented Decision-making
