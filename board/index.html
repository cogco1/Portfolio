术语 Glossary

真源 = docs/理论(记法) + 术语对照 + 提案卷 · 判决时更新

理论框架 Theory — 论文的骨头(记法以理论卷为准)

MDP · 马尔可夫决策过程Markov Decision Process

全项目的理论底座:建筑构成是一条决策轨迹 p(τ)=p(a₀…a_T|s₀),建筑只是这条轨迹的终点 B=f(τ)。与其只学习「在条件 C 下采样成品 B」,不如构造一个能够稳定推进、限制组合爆炸、记录承诺并支持局部修订的设计状态系统。「建筑不是被采样出来的,而是在连续承诺中成为现实。」 理论卷 §0

POMDP · 部分可观测 MDPPartially Observable MDP

真实设计天然包含隐藏意图。例如,「这里不能放柱子」背后的原因可能是尚未表达的视线要求、仪式序列或未来立面构想,因此设计过程天然是部分可观测的。ArchFlow 的对策不是要求模型自行猜测,而是把这些意图逐步显式化,见「构造马尔可夫性」。 理论卷 §3.3

构造马尔可夫性Constructed Markovianity

这不是发现设计过程天然符合马尔可夫性,而是主动把它构造成可操作的马尔可夫过程:将隐藏意图编译为显式状态变量,包括前提 premises:parti / function_intent / tradition;铁律11要求每个决策都必须读取这些前提,从而让「只看当前状态就能走下一步」成为工程事实。 premises.py · 理论卷 §3

状态编译 S_tState Compiler

将历史、提案与验证结果编译成最小充分状态 S_t=(G,C,L,O,U),并纳入义务账本 𝒪_t。完整思考历史容易重新激活噪声与过期假设,编译态则只保留会影响未来合法动作的信息。代码中的对应实体是黑板,包含槽位读写集与前提。 blackboard/ · 谱系:Hayes-Roth 1985 · Hearsay-II

熵预算Action Budgeter

将每一步的条件熵 H(A_t|S_t)≈log₂|A_valid| 控制在预算区间 [H_min,H_max] 内:掩码先把菜单缩到合法集 A_valid,策略再从合法项中选择。代码中的对应实体是 MenuSpec;目前已有三个真菜单接入流程。 policy.py · 谱系:SPRING(AIJ 2024) · shielded RL

D⁺(X) · 依赖闭包Dependency Graph

决策 X 的全部下游 D⁺(X)={Y|X⤳Y};修订成本 C_revision ∝ |D⁺(X)|——改一个决策的代价由依赖闭包决定,不由结果规模决定(repair locality 的测量面)。代码 = 读写集 + downstream_of()。 谱系:TMS(Doyle 1979) · de Kleer 1986

Selective Uncommit选择性修订 · P3.1

invalidate(D⁺(X)) → 局部重规划,而不是整图重生:只解除受影响的承诺,其余部分保持不变。它针对修改难以逆转的三个原因——下游扩散、编译丢史与路径依赖。施工序 P3.1 正在推进,这是全案风险最高的赌注,9 月设置检查点。 理论卷 §5–6

RAD · 检索增强决策Retrieval-Augmented Decision-making

这里使用 RAD 而非 RAG:RAG 把检索结果用于生成 token,RAD 把检索结果用于约束决策。它包含三部分:① 检索进入掩码文法,先例只影响「在合法项中选哪个」,不会引入非法动作;② 溯源记录,把影响每个决策的先例写入可回放的合法史;③ 通过确定性重放审计忠实度——删除检索信息后重放一次,观察选择是否改变。现有系统通常难以审计引用是否真正影响决策,而 ArchFlow 的重放基础设施提供了这一条件。当前仍待 Kevin 拍板入宪方式、语料池与指标,P2.5 已有接口桩。 提案_检索增强决策 · precedent.py · 谱系:CBR 三祖 · LeanDojo · LayoutGPT

义务 ⊕Obligation

义务是一种前向债:created → discharged,未清部分以 debt 留账。义务可以跨决策创建与清偿,例如 D05 创建隐私义务,D09 通过楼板检查清偿;cover→bear 则形成级联召唤。清偿按多重集差逐项对账,对应形式方法中的 proof obligation。 ledger · ⊕=比理论卷多出的增补件
